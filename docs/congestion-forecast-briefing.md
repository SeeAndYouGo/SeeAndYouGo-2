# 혼잡도 예측 기준 및 적용 결과 보고서

작성일: 2026-03-28
작성 목적: 서비스 운영팀, 개발팀, 기획팀 공용 검토 문서

## 1. 문서 개요

본 문서는 SeeAndYouGo 서비스에서 사용할 혼잡도 예측 방식을 정리한 보고서입니다. 이번 정리는 “복잡한 모델을 먼저 도입하는 것”보다 “현재 데이터로 가장 안정적으로 운영 가능한 기준을 먼저 확정하는 것”에 초점을 두고 진행하였습니다.

실제 운영 DB를 기준으로 여러 규칙을 비교한 결과, 현재 단계에서는 머신러닝 모델 없이도 충분히 설득력 있는 예측 체계를 구성할 수 있었습니다. 특히 개장 전 예측과 영업 중 예측은 성격이 다르므로, 하나의 방식으로 통일하기보다 각 상황에 맞는 규칙을 분리하여 사용하는 편이 더 적절하다고 판단하였습니다.

## 2. 예측 방식을 어떻게 설정하였는지

이번 예측 방식은 두 단계로 나누어 설계하였습니다. 첫 번째 단계는 아직 당일 실측 데이터가 충분히 쌓이지 않은 개장 전 예측입니다. 두 번째 단계는 영업이 진행되는 동안 들어오는 실제 값을 반영하는 실시간 예측입니다.

개장 전 예측에서는 “오늘이 과거 어느 날과 가장 비슷한가”를 판단 기준으로 삼았습니다. 단순한 전체 평균은 주말과 평일, 학기 중과 방학, 특정 요일의 고유 패턴을 제대로 반영하지 못하기 때문에 제외하였습니다. 대신, 가장 먼저 지난주 같은 요일 같은 시간대의 실제 값을 참고하고, 보조적으로 최근 8주 동안의 같은 요일 같은 시간대 평균을 반영하는 구조로 설정하였습니다.

영업 중 예측에서는 기준을 다르게 두었습니다. 영업이 시작된 이후에는 과거 평균보다 “방금 직전에 얼마였는가”가 훨씬 강한 신호로 작동하였습니다. 따라서 이 구간에서는 복잡한 보정보다 직전 5분 값을 중심으로 두는 편이 오차가 가장 작았습니다. 이때 “다음 5분 후 값이 이전 5분과 완전히 동일하다”는 가정을 두는 것은 아니며, 현재 시점에서 확인 가능한 정보 가운데 직전값이 가장 좋은 단일 추정치였다는 의미로 해석하시면 됩니다.

## 3. 비교 실험과 파라미터 선정 과정

파라미터 선정은 실제 데이터에 대해 전수 비교 방식으로 진행하였습니다. 테스트 구간은 2026년 3월 1일부터 2026년 3월 27일까지로 두었고, 5분 단위 슬롯을 기준으로 각 조합의 MAE, WAPE, RMSE를 비교하였습니다. 또한 이미 중복 적재나 이상 패턴이 확인된 날짜는 제외하였고, 슬롯 수가 과도하게 적은 날짜도 학습 대상에서 제외하였습니다.

개장 전 예측에서는 네 가지 구성요소의 가중치를 비교하였습니다. 기준이 된 값은 같은 요일 같은 시간대 평균, 최근 8주 같은 요일 같은 시간대 평균, 같은 메뉴 이력 평균, 그리고 지난주 같은 요일 같은 시간대의 실제 값이었습니다. 가중치는 0.0부터 1.0까지 0.1 간격으로 조합해 비교하였고, 메뉴 이력의 최소 사용 건수도 함께 바꾸어 검토하였습니다.

영업 중 예측에서는 직전 5분 값, 최근 15분 평균, 역사적 요일 평균, 최근 8주 평균을 서로 다른 비율로 섞었을 때의 성능을 비교하였습니다. 이 역시 동일하게 가중치 조합을 바꾸어 확인하였습니다.

## 4. 선정된 최종 파라미터

실험 결과, 개장 전 예측의 최적 조합은 사실상 “지난주 같은 요일 같은 시간대 값” 중심이었습니다. 정규화한 기준으로 보면 다음과 같은 비율이 가장 적절하였습니다.

| 구분 | 반영 비율 |
|---|---:|
| 지난주 같은 요일 같은 시간대 값 | 90.9% |
| 최근 8주 같은 요일 같은 시간대 평균 | 9.1% |
| 같은 요일 같은 시간대 전체 평균 | 0% |
| 같은 메뉴 이력 평균 | 0% |

이 결과는 두 가지 의미를 가집니다. 첫째, 주간 반복성이 매우 강하다는 점입니다. 둘째, 메뉴 exact match는 직관적으로는 좋아 보이지만, 현재 테스트 구간 전체를 기준으로 보면 기본 공식에 포함할 정도의 안정적인 개선 효과는 없었다는 점입니다. 따라서 메뉴는 기본 공식의 필수 요소가 아니라, 특정 식당 또는 예외 상황에서만 보조 신호로 활용하는 것이 더 적절합니다.

영업 중 예측의 경우에는 결과가 더 분명하였습니다.

| 구분 | 반영 비율 |
|---|---:|
| 직전 5분 값 | 100% |
| 최근 15분 평균 | 0% |
| 같은 요일 같은 시간대 평균 | 0% |
| 최근 8주 같은 요일 같은 시간대 평균 | 0% |

즉, 현재 운영 데이터에서는 실시간 예측의 최적 규칙이 매우 단순하게 정리되었습니다. 직전 5분 값이 존재하는 경우에는 그 값을 다음 5분 예측값으로 사용하는 것이 가장 안정적이었습니다.

## 5. 최종 적용 공식

위의 비교 결과를 바탕으로, 운영에 바로 적용할 수 있는 공식은 다음과 같이 정리할 수 있습니다.

개장 전 예측은 아래 공식을 기본으로 사용하시는 것이 적절합니다.

```text
예측값 = 0.909 × 지난주 같은 요일 같은 시간대 값
      + 0.091 × 최근 8주 같은 요일 같은 시간대 평균
```

다만 실제 운영에서는 모든 값이 항상 존재하지 않을 수 있으므로, 다음과 같은 fallback 순서를 함께 두는 편이 좋습니다.

```text
1순위: 지난주 같은 요일 같은 시간대 값
2순위: 최근 8주 같은 요일 같은 시간대 평균
3순위: 같은 요일 같은 시간대 평균
4순위: 같은 시간대 전체 평균
```

영업 중 예측은 아래처럼 두는 것이 가장 간결하고도 성능이 좋았습니다.

```text
예측값 = 직전 5분 값
```

직전값이 아직 존재하지 않는 첫 슬롯이나 예외 상황에서는 개장 전 예측값을 임시 대체값으로 사용하시면 됩니다.

## 6. 왜 이런 파라미터가 적절하다고 판단하였는지

개장 전 구간에서는 “오늘 토요일 12시의 혼잡도”를 예측할 때, 과거 전체 평균보다 “지난주 토요일 12시”가 훨씬 더 유사한 상황으로 작동하였습니다. 최근 8주 평균은 그 값을 약간 보정하는 용도로는 유효하였지만, 중심 기준이 되지는 못하였습니다. 다시 말씀드리면, 서비스의 혼잡도 패턴은 계절성이나 요일성도 중요하지만, 실제로는 주간 반복 구조가 가장 강하게 나타났습니다.

영업 중 구간에서는 이미 당일의 흐름이 시작되므로 상황이 달라졌습니다. 예를 들어 12시 30분에 혼잡도가 빠르게 오르고 있는 식당은 12시 35분에도 비슷한 수준을 유지하거나 소폭 증감하는 경우가 많았습니다. 이 구간에서는 역사 평균을 끌어오는 것보다 방금 직전 값을 사용하는 편이 훨씬 정확하였습니다. 따라서 실시간 예측의 핵심은 “오늘의 흐름을 그대로 반영하는 것”이라고 보시는 편이 맞습니다.

## 9. 운영 적용 시 권장 방식

실제 서비스에 적용하실 때에는 개장 전과 영업 중을 분리하여 운영하시는 편이 좋습니다. 개장 전에는 사전 예측값을 미리 계산해 저장해 두고, 영업이 시작되면 직전 실측값으로 계속 덮어쓰는 구조가 가장 자연스럽습니다. 이 방식은 계산 비용이 낮고, 설명 가능성이 높으며, 운영팀이 결과를 해석하기도 쉽습니다.

또한 메뉴 정보는 완전히 버리기보다, 현재 기본 공식의 일부로 넣지는 않되 예외 정책으로 남겨두는 편을 권장드립니다. 예를 들어 제1학생회관처럼 반복 메뉴의 영향이 큰 식당에서는 메뉴 보정 규칙을 별도 옵션으로 켤 수 있도록 설계하는 것이 좋겠습니다.

## 10. 최종 제안

현재 단계에서는 아래와 같은 방향으로 정리하는 것이 가장 합리적이라고 판단하였습니다.

첫째, 1차 운영 버전은 SQL 기반 규칙 예측으로 구축하시는 편이 좋습니다. 둘째, 개장 전 예측은 “지난주 같은 요일 같은 시간대 값”을 중심으로 두고 최근 8주 평균을 약하게 보정값으로 더하는 방식이 적절합니다. 셋째, 영업 중 예측은 “직전 5분 값”을 그대로 사용하는 단순 규칙이 가장 안정적입니다. 넷째, 메뉴는 기본 공식의 핵심 파라미터가 아니라, 특정 식당에 한해 별도 보정 규칙으로 관리하시는 편이 좋습니다.

이 문서의 기준으로 바로 구현을 진행하셔도 무리가 없으며, 이후 데이터가 더 쌓이면 그다음 단계에서 머신러닝 모델을 추가로 검토하시는 순서를 권장드립니다.
