# 혼잡도 예측 운영안 쉬운 설명 버전

작성일: 2026-03-28
검증 구간: 2026-03-01 ~ 2026-03-27
검증 단위: 식당별 5분 데이터

## 1. 이번에 정한 방법

혼잡도 예측은 하나의 규칙으로 모두 처리하지 않고, 예측하려는 시간 길이에 따라 다르게 보는 것이 더 잘 맞았습니다.

| 예측 시간 | 추천 방법 | 한 줄 설명 |
|---|---|---|
| 5분 후 | 지금 값 그대로 사용 | 아주 가까운 미래는 지금 상태가 거의 이어집니다. |
| 30분 후 | 30분 뒤 평소값 + 지금 차이의 80% 반영 | 조금 뒤는 평소 흐름과 현재 상황을 같이 봐야 합니다. |
| 60분 후 | 60분 뒤 평소값 + 지금 차이의 70% 반영 | 시간이 더 멀어질수록 평소 흐름 비중을 더 키워야 합니다. |

## 2. 왜 이렇게 나누었는가

| 구분 | 이유 |
|---|---|
| 5분 후 | 5분 정도는 사람이 갑자기 크게 바뀌지 않아 지금 값이 가장 강했습니다. |
| 30분 후 | 30분 뒤는 지금 값만 복사하면 오차가 커졌습니다. 해당 시간대의 평소 패턴을 같이 보는 편이 더 좋았습니다. |
| 60분 후 | 1시간 뒤는 현재 상황이 그대로 유지되지 않았습니다. 평소 패턴을 더 많이 반영해야 안정적이었습니다. |

## 3. 여기서 말하는 평소값

평소값은 "보통 이 시간에는 어느 정도 사람이 있었는가"를 뜻합니다.

| 참고 데이터 | 반영 비중 | 설명 |
|---|---:|---|
| 지난주 같은 요일, 같은 시간대 | 90.9% | 가장 최근의 비슷한 상황이라 가장 많이 반영했습니다. |
| 최근 8주 같은 요일, 같은 시간대 평균 | 9.1% | 지난주 값이 우연히 튀었을 수 있어 보조로 섞었습니다. |

예를 들어, 이번 주 토요일 12시 30분을 예측할 때는 지난주 토요일 12시 30분 값을 가장 많이 보고, 최근 여러 주의 같은 시간 평균은 조금만 보조로 사용합니다.

## 4. 실제 예측 방식

### 5분 후 예측

```text
예측값 = 지금 사람 수
```

### 30분 후 예측

```text
예측값 = 30분 뒤 평소값 + (지금 사람 수 - 지금 평소값) × 0.8
```

### 60분 후 예측

```text
예측값 = 60분 뒤 평소값 + (지금 사람 수 - 지금 평소값) × 0.7
```

## 5. 예시로 보면

현재 제1학생회관이 12시 기준 `100명`이고,
현재 시간의 평소값이 `80명`,
30분 뒤 평소값이 `120명`,
60분 뒤 평소값이 `90명`이라고 가정하겠습니다.

| 예측 시간 | 계산 방식 | 결과 |
|---|---|---:|
| 5분 후 | 지금 값 그대로 | `100명` |
| 30분 후 | `120 + (100 - 80) × 0.8` | `136명` |
| 60분 후 | `90 + (100 - 80) × 0.7` | `104명` |

즉, 30분 뒤와 60분 뒤는 단순히 지금 값을 복사하지 않고, 그 시간대의 평소 흐름으로 조금씩 돌아가도록 보는 방식입니다.

## 6. 실제로 얼마나 맞았는가

### 먼저 보는 기준

| 보는 항목 | 뜻 |
|---|---|
| 평균 차이 | 예측값이 실제값과 평균적으로 몇 명 차이 나는지입니다. 숫자가 작을수록 좋습니다. |
| `±10명 이내` 비율 | 예측값이 실제값과 10명 이내로 맞은 비율입니다. 숫자가 클수록 좋습니다. |
| 3단계 혼잡도 정확도 | `여유 / 보통 / 혼잡` 단계가 맞은 비율입니다. 숫자가 클수록 좋습니다. |

### 추천 방식 기준 결과

| 예측 시간 | 평균 차이 | `±10명 이내` 비율 | 3단계 혼잡도 정확도 |
|---|---:|---:|---:|
| 5분 후 | `4.531명` | `87.434%` | `94.915%` |
| 30분 후 | `11.738명` | `71.532%` | `86.504%` |
| 60분 후 | `15.585명` | `65.649%` | `82.669%` |

## 7. 최대치를 바로 제한하지 않은 이유

많이 몰린 시간대에는 "이제 곧 내려가야 하지 않을까"라는 생각이 자연스럽습니다. 다만 실제 데이터로 확인해보면, 5분 뒤 예측에서는 무조건 상한선을 씌우는 방식이 오히려 더 틀렸습니다.

| 비교 방식 | 평균 차이 | `±10명 이내` 비율 |
|---|---:|---:|
| 지금 값 그대로 사용 | `4.531명` | `87.434%` |
| 상한선(P95)으로 잘라서 사용 | `7.250명` | `84.165%` |

이유는 점심 피크 직전처럼 실제로는 조금 더 올라가야 하는 구간도 많기 때문입니다. 너무 일찍 "여기서부터는 내려갈 것"이라고 가정하면 오히려 예측이 나빠졌습니다.

정리하면, 최대치는 아예 안 보는 것이 아니라 `기본 예측값을 강제로 자르는 규칙`으로는 쓰지 않는 편이 좋았습니다. 대신 `너무 비정상적으로 튀는 값을 감지하는 참고 기준`으로 두는 것이 더 적절합니다.

## 8. 이 방식이 좋은 이유

| 항목 | 판단 |
|---|---|
| 이해하기 쉬운가 | 복잡한 모델 없이도 설명이 가능합니다. |
| 운영하기 쉬운가 | SQL이나 배치 로직으로 바로 만들 수 있습니다. |
| 결과가 충분히 괜찮은가 | 5분 후는 매우 안정적이었고, 30분 후와 60분 후도 단순 방식 중 가장 균형이 좋았습니다. |
| 팀 공유가 쉬운가 | "지금 상태 + 평소 흐름"이라는 개념만 이해하면 됩니다. |

## 9. 최종 제안

| 상황 | 적용 방식 |
|---|---|
| 영업 전 전체 시간대 예측 | 각 시간대의 평소값을 먼저 계산합니다. |
| 영업 중 5분 후 | 지금 값을 그대로 사용합니다. |
| 영업 중 30분 후 | 30분 뒤 평소값에 현재 차이를 80% 반영합니다. |
| 영업 중 60분 후 | 60분 뒤 평소값에 현재 차이를 70% 반영합니다. |
| 현재값이 없는 경우 | 해당 시간대의 평소값만 사용합니다. |
| 상한선 활용 | 예측값을 자르는 기본 규칙이 아니라, 이상치 점검용 참고값으로 사용합니다. |

## 10. 팀원 공유용 짧은 정리

이번 방식은 "아주 가까운 미래는 지금 상태를 그대로 보고, 조금 더 먼 미래는 그 시간대의 평소 흐름으로 조금씩 돌아간다"는 생각으로 이해하시면 됩니다. 최대치는 참고는 하되, 5분 뒤 예측을 무조건 잘라내는 규칙으로 쓰지는 않는 편이 더 잘 맞았습니다.
