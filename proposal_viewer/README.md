# 혼잡도 예측 독립 데모

SeeAndYouGo 본 프로젝트의 `backend`와 `frontend`를 수정하지 않고, 별도 화면에서 DB 기반 예측 결과를 확인하기 위한 작은 데모 프로젝트입니다.

현재 화면은 과거 데이터 검증용입니다. 식당과 날짜를 고르면 해당 날짜의 실제값과 예측값 차이를 `30분`, `60분`, `90분` 후 기준으로 비교합니다.

## 실행 방법

```bash
cd proposal_viewer
copy .env.example .env
npm install
npm start
```

브라우저에서 `http://localhost:4321`을 열면 됩니다.

주의: `public/index.html` 파일을 직접 열거나 기존 SeeAndYouGo 프론트 서버 주소에서 열면 `/api/history`가 HTML로 응답되어 `Unexpected token '<'` 오류가 날 수 있습니다. 반드시 `proposal_viewer`에서 `npm start`를 실행한 뒤 `http://localhost:4321`로 접속해야 합니다.

## 구조

- `server.js`: DB 조회 및 30분, 60분, 90분 후 과거 검증 API
- `public/index.html`: 식당, 검증 날짜, 파라미터를 입력하는 간단한 화면
- `public/app.js`: API 호출 및 결과 표시
- `public/styles.css`: 데모 화면 스타일

## 예측 공식

```text
예측값 = 목표 시점 평소값 + (현재값 - 현재 시점 평소값) * 현재 차이 반영률
```

예를 들어 60분 뒤 평소값이 45명, 현재값이 80명, 현재 시점 평소값이 60명, 60분 반영률이 0.4라면 다음과 같습니다.

```text
45 + (80 - 60) * 0.4 = 53명
```

## 기본 파라미터

| 식당 | 30분 | 60분 | 90분 | 평균 기준 |
|---|---:|---:|---:|---|
| 생활과학대 | 0.0 | 0.0 | 0.0 | 시즌성 평균 |
| 학생생활관 | 0.2 | 0.1 | 0.0 | 전 기간 평균 |
| 그 외 식당 | 0.6 | 0.4 | 0.2 | 시즌성 평균 |

화면에서 `30분 반영률`, `60분 반영률`, `90분 반영률`을 직접 바꿀 수 있습니다. 값이 클수록 현재 혼잡도가 예측값에 더 많이 반영되고, 값이 작을수록 과거 평균인 평소값을 더 강하게 따릅니다.

## 검증 API

```http
GET /api/history?restaurant=제1학생회관&date=2026-03-18&w30=0.6&w60=0.4&w90=0.2
```

응답에는 각 예측 범위별 `MAE`, `10명 이내 비율`, 비교 건수, 시간대별 실제값과 예측값이 포함됩니다.

## 보안 메모

DB 주소와 계정은 브라우저 코드에 넣지 않고 서버의 `.env`에서만 관리합니다. 실제 배포 시에는 `.env`를 커밋하지 말고, 서버 환경변수나 Secret Manager로 관리하는 편이 안전합니다.
